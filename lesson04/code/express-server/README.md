RESTful API - это архитектурный стиль взаимодействия между приложениями.
Он основан на протоколе HTTP. То есть описание правил обмена данными между клиентом и сервером

1. Resources
- URL - Uniform Resource Locator (путь по которому находится необходимый ресурс)

Используются существительные (в ед или множ числе)
Не используется глаголы!

/users - получить данные всех пользователей
/users/12342 - получение данных пользователя по id или
               удаление пользователя с id

Переменные пути        
/users/{id}
/users/:id 

/getAllUsers - для путей это плохой нейминг
/users/18/posts/14  - излишняя вложенность

Самый лучший стиль наименования путей - kebab case
/user-profile


query params
url?key=value&key=value

2. Клиент-сервер
METHODS
CRUD - Create Read Update Delete
C - POST /users - создание нового пользователя
R - GET /users - получение всех пользователей
U - PUT /users/:id - отредактировать данные пользователя
    PATCH /users/:id - частично отредактировать пользователя
D - DELETE /users/:id - удаление пользователя

STATUS CODE
100-199 - info
200-299 - success (201 - created)
300-399 - redirect
400-499 - client error
500-599 - server error

3. Stateless - отсутствие состояния. Сервер не хранит данные о запросах, которые
были отправлены. Каждый запрос должен иметь всю необходимую информацию

4. Единообразие 
Вся информация должна быть передана в общем формате - JSON
